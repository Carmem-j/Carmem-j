ROI No-Code

Objetivos

  • Integração de features de visualização de dados para o principal widget no-code de dados temporais da plataforma, pois o widget de referência é híbrido e alguns recursos apenas são possíveis via código;
  • Substituição completa do widget usado atualmente, pela complexidade técnica e alto custo de manutenção e correção de bugs;
  • Permitir o uso das features originárias do widget usado em fracking para outros mercados.

2. Desafio

A construção do mapa de stakeholders no início do processo foi essencial para identificar todos os atores importantes do projeto e agrupá-los para uma melhor gestão das partes interessadas, de acordo com a fase do projeto em relação à tríade: tecnologia, negócios ou usuário.
